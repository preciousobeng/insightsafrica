## 1. Why this task, and why first

SPI-3 is the first module of the Flood Early-Warning Suite. It is deliberately first because:

- It has an **objectively correct answer**, so the acceptance tests can be definitive (see section 6, Test A).
- It **unblocks** the Risk Index, the seasonal outlook, and SPI-comparability across regions with different climatologies.

This task is data-layer computation plus tests only. **API exposure and any frontend are explicitly out of scope** for this sprint (see section 8).

## 2. What SPI-3 actually is (do not implement a z-score)

SPI is **not** (x − mean) / std. That is a standardized anomaly, which we already have as the z_score field in the anomaly files. SPI is the McKee et al. (1993), WMO-standard transform:

1. For each area, form the **3-month rolling precipitation sum**. SPI-3 for month *M* = rainfall(*M*) + rainfall(*M*−1) + rainfall(*M*−2), spanning the year boundary (SPI-3 for Feb 2024 = Dec 2023 + Jan 2024 + Feb 2024).
2. For the target **calendar month** *M*, gather the set of those 3-month sums ending in *M* across the **1991–2020 reference period** (n is about 30).
3. Fit a **gamma distribution with zero-handling** (Thom) to that reference set:
   - q = P(zero) = (count of zero sums) / n
   - fit gamma (shape alpha, scale beta) to the non-zero values, location fixed at 0
   - cumulative dist: H(x) = q + (1 − q) · GammaCDF(x; alpha, beta) for x greater than 0; H(0) = q
4. **SPI = inverseNormalCDF(H(x))** — the inverse standard-normal CDF of that cumulative probability.
5. **Clamp** SPI to the range [-3.09, 3.09] to avoid plus/minus infinity at distribution extremes.

Use scipy.stats.gamma (fit with floc=0) and scipy.stats.norm.ppf.

> Flood framing: SPI is usually drought-focused, but the **wet tail (SPI at or above +1)** is exactly what flood risk needs. Keep both tails; we lead on the wet side.

## 3. Inputs (existing, do not change their format)

- **Monthly rainfall:** data/archive/{country}/stats/chirps-v2.0.{YYYY}.{MM}_{country}.json
  schema: { "year", "month", "zonal_stats": { level: { area: { "mean": <mm>, ... } } } }
  levels are e.g. regions, districts; area keys are "Ashanti" (region) and "Kumasi|Ashanti" (district, name-pipe-region).
- **Reference period:** 1991–2020, matching compute_ltm_baseline.py.

Build the contiguous per-area monthly series by reading **all** stats files (not just the reference window — you need *M*−1 and *M*−2, which may fall in a prior year). A 3-month sum is only valid if all three months are present; otherwise emit null for that area-month and record it as skipped.

## 6. Acceptance tests (deliver tests/test_spi.py; all must pass)

These test **behaviour and correctness**, not "it runs." A module that runs but computes a z-score fails Test A.

- **A — Reference-period normality (the definitive correctness check).** By construction, the SPI-3 values of the 1991–2020 reference set, per district per calendar month, must be approximately standard normal: **absolute mean below 0.10** and **std between 0.85 and 1.15**. If the transform is wrong (e.g. a z-score, or a mis-fit gamma), this fails. Run across all Ghana districts by 12 months.
- **B — Range and completeness.** Every SPI value is finite, within [-3.09, 3.09], no NaN/inf. Every area present in the source stats for that month is present in the SPI output (or explicitly null with a skip reason).
- **C — Independent cross-check.** For at least 5 randomly chosen district-months, compute SPI-3 with an *independent* reference implementation (the standard_precip or climate_indices package, **or** a self-contained scipy reference coded inside the test) and assert agreement within **plus/minus 0.05**. Catches "looks like SPI but isn't."
- **D — Zero-handling.** A district/month whose reference set contains zero 3-month sums (dry-season cases) still yields a finite SPI — no gamma-fit crash, p_zero reflected in fit.
- **E — No look-ahead.** Computing SPI for month *M* with later months absent from the archive yields identical output to computing it with later months present. (Prove no future leakage.)
- **F — Determinism.** Two consecutive runs produce byte-identical files.
- **G — Hindcast sanity (LOCKED 2026-06-29).** SPI-3 measures rainfall excess over a 3-month window, **not** flood occurrence — so this test asserts only that events the historical record documents as *exceptional rainfall* show an elevated SPI-3 signal. It deliberately does **not** claim SPI flags every flood: single-day cloudbursts get smoothed by the 3-month sum, and drainage-failure floods happen on normal rain (those are the later Risk Index's job, not SPI's). Assert on the **region-level "Greater Accra"** SPI-3 value (coarser = less district noise, and matches how the sources report "Accra"). **Note the region key in the stats files is `GreaterAccra` (no space)** — index it by the actual key, not the display name. The 3-month window ends in the event month.
  - **G1 (wet):** window ending **1995-07** (about 50-year return-period rainfall) gives SPI-3 at or above **+1.0**
  - **G2 (wet):** window ending **2014-06** (10+ hour sustained downpour) gives SPI-3 at or above **+1.0**
  - **G3 (wet, our live data):** window ending **2024-05** (221.3 mm, wettest single month in the live record; out-of-reference period) gives SPI-3 at or above **+1.0**
  - **G4 (soft / informational):** window ending **2015-06** (the 3 June disaster — but largely a single-day cloudburst, so a 3-month sum may be only mildly elevated) gives SPI-3 at or above **0.0**. Strongly positive is good; near zero is expected, not a failure — comment it.
  - **G5 (dry side — sign-consistency, no fabricated event):** for every reference-period area-month whose 3-month sum is **below** its fitted distribution median, SPI is negative; **above** median, SPI is positive. Guarantees the dry tail behaves correctly without asserting a specific drought.
  - NADMO's district-level records (roadmap #11, via the institutional ask) would later upgrade G into rigorous per-district calibration.
